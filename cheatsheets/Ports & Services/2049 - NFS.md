# TCP 2049 (NFS)

## Overview

During internal network enumeration, port 2049 is associated with NFS, which is used for file sharing between systems. If misconfigured, it may expose shared directories that can be mounted remotely, potentially revealing sensitive files or allowing privilege escalation depending on export permissions. Misconfigurations such as weak export rules can significantly increase exposure.

---
Allows file system access over a network as if they were local, same purpose as SMB. NFS usually runs on ports 111 and 2049

If we discover an NFS service with nmap we can mount it on our machine. We create an empty NFS share to mount it to, then we can navigate the file system

Show available shares
```
showmount -e 10.129.14.128
```

Mounting
```
mkdir target-NFS
sudo mount -t nfs 10.129.14.128:/ ./target-NFS/ -o nolock
cd target-NFS
tree .
```

Listing Contents with username and UIDS
```
ls -l mnt/nfs/
ls -n mnt/nfs/
```

Unmounting
```
cd ..
sudo mount ./target-NFS
```
